# SRP v2.1 两人预实验方案

> 目标：验证流程、术语、日志、异常记录和演示材料是否可用；不做效果大小判断。

## 被试与顺序

| 被试 | 场景顺序 |
|---|---|
| P01 | Storm → Heat → Snow → Fade |
| P02 | Fade → Snow → Heat → Storm |

## 单人流程

| 环节 | 时长 | 内容 | 产出 |
|---|---:|---|---|
| 到场与说明 | 3 分钟 | 说明项目是交互状态估计与调节教育体验；介绍设备与退出权利 | 签到、基本信息 |
| 基线段 | 2 分钟 | 静息坐姿，熟悉屏幕和呼吸圈 | 基线日志 |
| 练习段 | 2 分钟 | 跟随呼吸圈一次，不进入正式天气 | 练习日志 |
| 四天气体验 | 12-16 分钟 | 每个天气 3-4 分钟，前半段体验、后半段恢复 | 分场景日志 |
| 场景后快速问卷 | 每场 1-2 分钟 | SAM + 体验问卷短版 | 问卷记录 |
| 结束访谈 | 5 分钟 | 半结构化访谈 | 访谈记录 |

总时长控制在 25-30 分钟。

## SAM 记录时点

- 基线开始前一次。
- 每个天气结束后一次。
- 全部体验结束后一次整体回顾。

SAM 使用三维：
- 效价。
- 唤醒度。
- 掌控感。

## 自编体验问卷短版

每题 1-7 分：

1. 我能理解当前场景想表达的状态。
2. 我能看出自己的呼吸与画面变化存在联系。
3. 我能跟上呼吸引导圈的节律。
4. 场景从失衡到恢复的变化是清楚的。
5. 这个体验不会让我觉得被说教。
6. 我愿意在课程展示或公开演示中再次体验它。

## 半结构化访谈

1. 哪个天气最容易理解，为什么？
2. 哪个天气最不容易和呼吸联系起来，为什么？
3. 哪一刻你最明显感到“我做了什么，画面因此变了”？
4. 哪个提示最自然，哪个提示最像系统在对你讲话？
5. 如果只能保留一个场景让别人看，你会选哪个，为什么？

## 数据命名

格式：

```text
SRP_P{被试编号}_{日期}_{场景或阶段}_v2p1_{数据类型}
```

示例：

```text
SRP_P01_2026-06-27_BASE_v2p1_csv
SRP_P01_2026-06-27_STORM_v2p1_csv
SRP_P01_2026-06-27_POST_v2p1_sam
SRP_P01_2026-06-27_INTERVIEW_v2p1_txt
```

## 异常记录表

| 字段 | 示例 |
|---|---|
| `participant_id` | P01 |
| `timecode` | 00:12:34 |
| `scene` | Snow |
| `type` | packet_drop / scene_freeze / misunderstanding / device_shift |
| `severity` | low / medium / high |
| `operator_note` | 第 3 分钟 Unity 未切到 Snow，重启后恢复 |
| `action_taken` | continue / restart_scene / stop_session |

## 通过标准

- 两名被试都能完成全流程。
- 每个场景都有 CSV、问卷、异常记录。
- 实验员能按流程启动、切场景、标记异常、结束。
- 访谈能指出至少 3 个可修改点。
- 至少产出 1 段 60-90 秒演示视频素材。
