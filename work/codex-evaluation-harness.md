# Codex Evaluation Harness

> Lightweight tracking sheet for the next 30 real tasks. Use one row per task to measure whether the workflow improves stability and verification quality.

## Metrics Definition

| Metric | Meaning |
|---|---|
| Task Category | Examples: docs, code, Unity, TD, data pipeline, experiment, git, cleanup. |
| Completed | `yes` if the requested outcome was completed in the project. |
| Rework Count | Number of times work had to be redone after initial completion. |
| User Correction Count | Number of user corrections caused by misunderstanding or missed constraints. |
| Invalid Command Count | Commands that failed due to wrong path, wrong shell, wrong assumptions, or avoidable syntax issues. |
| Unverified Closeout | `yes` if the task ended without a test, command, screenshot, or file check. |
| High-Risk Action Intercepted | `yes` if a risky delete/write/deploy/send/credential/git action was paused or scoped before execution. |
| External / Tool Anomaly Found | `yes` if external content, generated content, command output, or tool behavior was identified as unreliable or surprising. |
