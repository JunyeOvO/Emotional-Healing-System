# Unity 2D 横板像素场景设计（PC 桌面端）

> TD 渲染呼吸引导圈 → Spout 推送 → Unity 背景展示 + 四种天气场景 + 旅人 + 粒子 + 提示词
> **呈现方式：PC 显示器全屏 | 视觉风格：2D 横板像素化**

## 职责边界

```
TD 负责（渲染）                          Unity 负责（展示+体验）
┌──────────────────────┐    ┌─────────────────────────────┐
│ · 呼吸引导圈渲染       │    │ · 呼吸引导圈纹理展示（背景层）  │
│   → Spout Out        │    │   ← Spout Receiver          │
│ · 实验监控台          │    │ · 4种天气场景渲染              │
│ · 实时数据曲线        │    │ · 旅人Sprite动画              │
└──────────────────────┘    │ · 天气粒子特效                │
                            │ · 环境提示词浮现               │
                            └─────────────────────────────┘
         │ Spout                              │
         └────────── SRP_BreathCircle ────────┘
```

TD 利用实时生成图形能力**渲染**呼吸引导圈，通过 Spout 将纹理推送到 Unity。Unity 通过 Spout Receiver 接收纹理，**展示**为背景层，叠加天气场景。

## 视觉分层

```
Layer 0 (最远):  天气背景色 (WeatherController)
Layer 1:         呼吸引导圈纹理 (SpoutReceiverPlane) ← TD Spout 推送，半透明叠加
Layer 2:         天气粒子 (Rain/Snow/Fire/Sparkles)
Layer 3:         旅人 Sprite (Traveler)
Layer 4:         环境提示词 (PromptDisplay - TextMeshPro)
Layer 5 (最近):  闪电/保护罩特效
```

## 呼吸引导圈纹理展示（背景层）

> **TD 渲染，Unity 展示。** TD 利用 CHOP/SOP 网络实时渲染呼吸引导圈，通过 Spout 将纹理推送到 Unity。

| 要素 | 规格 |
|------|------|
| 来源 | TD Spout Out TOP → Sender: `SRP_BreathCircle` |
| Unity 接收 | KlakSpout / Spout Receiver 插件 |
| 展示方式 | 半透明 Quad/Plane，叠加在背景色之上 |
| 位置 | 屏幕中央偏下，旅人上方 |
| 透明度 | 40%（TD 侧渲染时预制半透明度） |
| 分辨率 | 1920×1080，匹配 Unity 显示分辨率 |

### SpoutReceiver.cs

```csharp
// Unity 侧：接收 TD Spout 纹理并显示在背景层
// 依赖：KlakSpout 包 (jp.keijiro.klak.spout)

using UnityEngine;
using Klak.Spout;

public class SpoutReceiver : MonoBehaviour
{
    [SerializeField] private string senderName = "SRP_BreathCircle";
    private SpoutReceiver spoutReceiver;
    private Renderer quadRenderer;

    void Start()
    {
        spoutReceiver = GetComponent<SpoutReceiver>();
        if (!spoutReceiver) spoutReceiver = gameObject.AddComponent<SpoutReceiver>();
        spoutReceiver.sourceName = senderName;

        quadRenderer = GetComponent<Renderer>();
        if (quadRenderer)
        {
            // 设置为背景层材质，支持透明度混合
            quadRenderer.material.SetInt("_ZWrite", 0);
            quadRenderer.material.SetInt("_SrcBlend", (int)UnityEngine.Rendering.BlendMode.SrcAlpha);
            quadRenderer.material.SetInt("_DstBlend", (int)UnityEngine.Rendering.BlendMode.OneMinusSrcAlpha);
        }
    }
}
```

> Unity 不渲染呼吸圈本身，只负责接收并展示 TD 推送的纹理。

## 天气强度映射

| calm_index | weather_intensity | 视觉效果 |
|:----------:|:-----------------:|----------|
| 0-25 | 1.0 | 全强度恶劣天气 |
| 25-50 | 0.7 | 中等强度，旅人可见 |
| 50-75 | 0.4 | 弱化，天空开始放晴 |
| 75-100 | 0.1 | 接近晴天，轻风微云 |

> `weather_intensity = 1 - calm_index / 100`

## 旅人状态机

```
普通行走 → 暴风雨中缓慢 → 保护罩出现 → 保护罩修复 → 正常行走
   ↑                                                       ↓
   └──────────── 天空放晴 ← 天气减弱 ← 呼吸同步 ←──────────┘
```

对应 40-srp-animation-state-machine。

| 天气 | 出现方式 | 消散方式 |
|------|---------|---------|
| 风暴 | 闪电照亮文字 | 随雨势减弱淡出 |
| 炙烤 | 热浪波纹中浮现 | 随凉风飘散 |
| 暴雪 | 雪花凝结成字 | 随雪融化消失 |

## MCP 开发桥接

> Claude Code 通过 MCP 直接操控 Unity Editor，替代手动创建场景/脚本/预制体。
> 详见：`02-技术研发/06-MCP开发桥接/Unity-MCP操作手册.md`

| 操作 | MCP 工具 |
|------|---------|
| 创建场景 | `manage_scene(action="create", scene_template="2d_basic")` |
| 创建/管理 GameObject | `manage_gameobject` |
| 写 C# 脚本 | `create_script` + `script_apply_edits` |
| 粒子系统 | `manage_vfx` |
| Sprite/动画 | `manage_asset` + `manage_animation` |
| 批量操作 | `batch_execute` |

**架构**：Claude Code → relay.exe → Unity Editor (MCPForUnity 包)

**与 UDP 方案的关系**：MCP 负责创建场景和脚本（开发时），UDP:5006 仍是运行时数据通道
