# SRP v2.1 下一阶段 7 天推进计划

> 推进原则：先锁定可验证闭环，再补场景机制，最后进入预实验与交付材料整理。

## 核心判断

下一阶段不扩展更复杂的识别逻辑。四天气先作为实验条件或用户选择入口，Python 生理/模拟输入负责驱动天气恢复强度、视觉强度与呼吸引导同步性。

本周必须锁定四件事：

1. Python 3.14 环境和 pytest 回归入口。
2. UDP v1.2 字段契约。
3. 同机 TD → Spout → Unity 条件。
4. 四天气最小验收标准。

## 四天气最小验收

| 场景 | 核心机制 | 输入字段 | 验收方法 |
|---|---|---|---|
| Storm | 风雨强度衰减 | `weather.type=storm`、`weather.intensity`、`calm_index` | 从高强度到低强度，雨量/闪电明显下降 |
| Heat | 热浪扭曲衰减 | `weather.type=heat`、`weather.intensity`、`calm_index` | 画面由灼热刺眼过渡到可停留 |
| Snow | 遮蔽度衰减 | `weather.type=snow`、`weather.intensity`、`calm_index` | 雪雾降低，路径重新可见 |
| Fade | 去饱和恢复 | `weather.type=fade`、`weather.intensity`、`calm_index` | 画面从灰白压平恢复到有层次的颜色世界 |
