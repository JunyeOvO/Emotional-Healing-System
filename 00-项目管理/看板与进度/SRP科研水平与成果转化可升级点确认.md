# SRP 科研水平与成果转化可升级点确认

> 依据：2026-06-24 完整阅读《在不改变底层架构前提下提升大学项目科研水平与高价值成果转化的规划报告》，并对照当前 SRP v2.1 项目文档、数据链路、实验方案和质量门禁。

## 1. 核心判断

SRP 当前已经具备从“工程原型”升级为“可发表、可复现、可交底、可展示”的基础，但升级重点不应放在更复杂的识别模型或重写底层系统上。

下一阶段最值得做的是：

1. 把现有系统表述从“生理数据驱动天气画面”升级为“可穿戴交互状态估计驱动的情绪调节教育反馈任务”。
2. 把实验从“体验后主观反馈”升级为“主观量表 + 四维评分趋势 + 事件日志 + 复现实验”的证据链。
3. 把成果从“一个结题系统”拆成论文、知识产权、软著、数据样例、演示视频和复现包的组合。
4. 把质量门禁从工程可运行扩展到科研可用，包括字段冻结、数据完整性、统计分析、复现材料和术语合规。

## 2. 当前项目可支撑的升级基础

| 基础资产 | 当前状态 | 对升级的意义 |
|---|---|---|
| Python 数据链路 | 已形成 mock/设备输入、四维评分、UDP v1.2、CSV 日志、测试体系 | 可支撑系统论文、方法说明、数据复盘和软著材料 |
| UDP 字段冻结表 | 已明确 Python/TD/Unity/CSV 的统一字段契约 | 可作为复现包和系统交付的核心规范 |
| Unity 四天气原型 | Storm/Heat/Snow/Fade 场景文件存在，非风暴机制仍需验收 | 可支撑 HCI/交互设计方向论文和演示视频 |
| TouchDesigner 职责 | 呼吸引导圈、实验监控台、Spout 输出定位清楚 | 可支撑主试监控、数据可视化和联调证据 |
| 实验设计 | 8-12 名大学生，被试内设计，SAM + 自编问卷 + 访谈 | 已具备低成本可执行的研究设计雏形 |
| 质量门禁 | G1-G7 已覆盖功能、性能、术语、实验和交付 | 可升级为科研质量门和成果筛选门 |
| 文献包 | 已有多类参考文献 | 可支撑问题定义、相关工作和讨论章节 |

## 3. A 类：立即可升级点

这些升级不需要改底层架构，主要靠补文档、补测试、补实验记录和补证据。

| 可升级点 | 当前抓手 | 升级动作 | 目标成果 |
|---|---|---|---|
| A1 研究问题重构 | 四天气 + 四维评分 + 呼吸反馈闭环 | 定义“可穿戴交互状态估计驱动的天气化反馈任务”，明确输入、状态、反馈、恢复趋势四要素 | 论文核心问题、结题报告主线 |
| A2 证据链强化 | CSV、SAM、访谈、四维评分 | 建立每场景 7 分钟的固定分析模板：趋势图、前后变化、异常事件、主观反馈摘要 | 实验章节、图表包 |
| A3 数据治理升级 | UDP v1.2、CSVLogger、实验数据目录 | 固定 CSV 列、命名、被试编号、异常标记、备份规则；生成一份样例数据字典 | 复现包、数据附录 |
| A4 方法学质量门 | SE-G0 到 SE-G5、G5/G6 | 增加对照条件、场景顺序随机、负例记录、复现实验责任人 | 投稿前质量门 |
| A5 可复现运行包 | README_run、requirements、Unity 工程 | 形成“从 mock 到 60 秒闭环”的一键复现实验记录，包括命令、日志、截图、CSV | 软著、答辩、系统论文附录 |
| A6 成果漏斗 | 现有 G7 交付清单 | 为每个成果建立 A/B/C 分级卡，按新颖性、证据强度、可复现性、可发表性、可交底性评分 | 管理台账 |
| A7 术语合规升级 | 质量门禁术语清单 | 把术语扫描纳入 G5/G7；用户可见文本统一为“交互状态估计”和“情绪调节教育” | 降低表达风险 |

## 4. B 类：补齐证据后升级点

这些方向有价值，但需要先完成联调、预实验或外部评审，否则容易停留在概念层。

| 可升级点 | 缺口 | 补齐条件 | 预期产出 |
|---|---|---|---|
| B1 HCI/交互体验论文 | 需要完整 4 天气流程和用户数据 | 完成 2 人预实验 + 8-12 人正式实验，形成量表与访谈编码 | 应用型会议/期刊论文 |
| B2 系统论文 | 需要端到端稳定性证据 | 60 秒、10 分钟、40 分钟三档运行记录，TD/Unity 同步截图 | 系统设计论文、结题技术报告 |
| B3 可穿戴数据链路方法稿 | 需要设备降级证据 | mock、无 EDA、无 Polar H10、无呼吸带四类回退测试 | 工程方法论文、技术交底书 |
| B4 视觉反馈机制研究 | 需要四天气各自唯一核心机制稳定 | 每种天气只保留一个主机制，并做前后对比图 | 设计研究稿、演示视频 |
| B5 知识产权组合 | 需要先做新颖性检索和公开边界 | 完成成果披露表、技术交底书和初步检索 | 发明交底、软著材料、界面/视觉保护预案 |
| B6 外部合作材料 | 需要标准化展示包 | 准备 3 分钟视频、1 页系统卡、数据链路图、风险边界说明 | 校内外路演、合作沟通 |

## 5. C 类：暂不建议升级点

| 方向 | 暂缓原因 | 建议替代 |
|---|---|---|
| 更复杂的情绪自动分类 | 当前项目约束强调教育交互，不适合把消费级数据用于严肃判断 | 保持四天气为实验条件或用户选择入口 |
| 重写通信架构 | UDP v1.2 已能支撑 10Hz 同机闭环 | 先做字段冻结、丢帧统计和降级策略 |
| 增加大量新设备 | 会放大采集不稳定和工期风险 | 优先 mock + Polar H10 + 呼吸带，EDA 可做可选通道 |
| 大规模人群实验 | 当前周期到 6/30，资源不足 | 先完成 2 人预实验和 8-12 人小样本可用性实验 |
| 追求跨平台部署 | 目前目标是同机 Windows 闭环 | 先保证本机复现，再考虑打包迁移 |

## 9. 新增质量门建议

| 门禁 | 检查项 | 通过标准 |
|---|---|---|
| RQ-G1 研究问题门 | 任务定义是否清楚 | 能用一句话说明输入、反馈、目标和边界 |
| EV-G1 证据链门 | 数据、问卷、访谈是否能互相解释 | 每个天气至少有一组趋势图和一段用户反馈摘要 |
| REP-G1 复现门 | 外部成员能否按文档运行 | 按 README_run 完成 60 秒 mock 闭环 |
| IP-G1 公开边界门 | 是否存在应先交底的成果 | A 类成果公开前完成一次检索和交底判断 |
| TERM-G1 术语门 | 是否符合项目表达边界 | 用户界面、报告、PPT、论文草稿统一使用项目推荐术语 |

## 10. 结论

SRP 当前最有价值的升级路线是“证据链 + 复现包 + 双轨成果”，而不是继续扩大识别复杂度。只要在 6/24-6/30 内完成闭环证据包、2 人预实验、成果漏斗和第一份技术交底提纲，项目就能从普通课程式工程展示升级为更接近高校科研评价标准的组合成果。
